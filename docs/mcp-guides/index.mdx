Whether you're debugging issues, creating new features, or optimizing your development workflow, MCP provides a natural language interface to DevCycle's powerful feature management platform.

## Quick Examples

### Creating a Feature Flag

```
You: "Create a feature flag for the new dashboard layout"

AI: I'll create a feature flag called 'new-dashboard-layout' with:
- Boolean variable 'enabled' (default: false)
- Available in all environments
- Targeting disabled initially
- Ready for gradual rollout
```

### Managing Environments

```
You: "Enable the mobile-redesign flag in staging with 50% rollout"

AI: I've enabled 'mobile-redesign' in staging environment:
- Targeting: 50% of users
- Status: Active
- Ready to test with your team
```

### Debugging Issues

```
You: "Why isn't the new-feature flag working in production?"

AI: Analyzing 'new-feature' flag in production:
- Status: Active ✅
- Targeting: Only users with email ending in @company.com
- Issue: Your test user doesn't match the targeting rules
```

## Best Practices

### 1. **Use Descriptive Requests**

Be specific about what you want to achieve:

- ❌ "Create a flag"
- ✅ "Create a feature flag for the new checkout flow with a boolean toggle"

### 2. **Specify Environments**

Always mention which environment when making changes:

- ❌ "Enable the flag"
- ✅ "Enable the checkout-redesign flag in staging"

### 3. **Confirm Production Changes**

The AI will ask for confirmation before making production changes:

- Production flag modifications require explicit approval
- Staging and development changes are immediate

### 4. **Use Natural Language**

Describe your intent rather than technical specifics:

- ✅ "Set up A/B testing for 20% of mobile users"
- ✅ "Gradually roll out to premium customers first"
- ✅ "Turn off the experimental feature for everyone"

## Troubleshooting

### Common Issues

**"I don't have access to DevCycle tools"**

- Verify MCP server is running: `dvc mcp status`
- Check client configuration includes DevCycle server
- Restart your AI client after configuration changes

**"Permission denied when updating flags"**

- Verify API key has required permissions
- Check if you have access to the target environment
- Production changes may require additional permissions

**"Flag changes aren't taking effect"**

- Check the environment you're targeting
- Verify targeting rules are configured correctly
- Allow up to 1 minute for changes to propagate
